# Gem Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Clean up the `commit-context` gem so it builds, installs, and loads correctly as a dependency in other Ruby projects.

**Architecture:** The repo is pre-initial-commit. The staged index still contains the old `lib/commit/` namespace (bundler's default scaffold), while the working tree has been refactored to a flat `lib/commit_context/` namespace. `commit-context.gemspec:19` computes `spec.files` from `git ls-files`, so untracked files would be silently excluded from the built gem. This plan clears the stale scaffold, tracks the new files, fixes a stale `require` in `bin/console`, adds a license, and verifies `gem build` + install-and-require from a clean directory.

**Tech Stack:** Ruby 3.2+, Bundler, RubyGems, Minitest.

---

No source-code behavior changes — this is purely a packaging fix.

### Task 6: Verify the gem installs and loads in an isolated environment

This simulates what a downstream project experiences. A gem can `build` successfully and still be broken at `require` time (missing files, namespace mismatches).

- [ ] **Step 1: Build, install into a scratch prefix, and require**

```bash

gem build commit-context.gemspec
mkdir -p /tmp/commit-context-smoke
gem install --install-dir /tmp/commit-context-smoke --no-document commit-context-0.1.0.gem
GEM_PATH=/tmp/commit-context-smoke GEM_HOME=/tmp/commit-context-smoke ruby -e 'require "commit_context"; puts CommitContext::VERSION; puts CommitContext.constants.sort'
```

Expected:
- `gem install` completes with `Successfully installed commit-context-0.1.0` and `1 gem installed`.
- Ruby prints `0.1.0` followed by a sorted constants list including `CLI`, `Git`, `Report`, `Sections`, `TruncatedDiff`, `VERSION`.
- No `LoadError`, no `NameError`.

- [ ] **Step 2: Verify the installed executable runs**

```bash
GEM_PATH=/tmp/commit-context-smoke GEM_HOME=/tmp/commit-context-smoke /tmp/commit-context-smoke/bin/commit_context --help 2>&1 | head -20
```

Expected: CLI help output (or at minimum: the process starts without `LoadError`/`NoMethodError`). If the CLI doesn't support `--help`, substitute whatever invocation `lib/commit_context/cli.rb` supports; the goal is to prove the executable boots through its `require` chain.
